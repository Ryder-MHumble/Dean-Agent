# 院长智能体 · 系统信息架构设计方案

> **面向高校 AI 学院院长的智能决策支持系统**
> 版本：v1.0 | 日期：2024年1月 | 状态：详细设计阶段

---

## 📋 方案概述

本方案设计了一套**从领导视角出发、以行动为导向**的院长智能决策支持系统。通过 **8 个核心模块** + **右下角智能助手** + **钉钉深度集成**，实现：

- ✅ **信息降噪**：每天只看最重要的 3-5 件事，优先级智能评分过滤
- ✅ **一键行动**：审批、催办、发送、生成，全流程钉钉集成，无需切换系统
- ✅ **移动优先**：PC 端与钉钉实时同步，随时随地决策
- ✅ **AI 辅助**：主动预测内参机会、人才信号、关系维护、日程冲突
- ✅ **维度纯净**：27 个子页面严格分离，不混杂学生、经费、项目等信息

**核心价值：让院长每天节省 2 小时，决策效率提升 50%**

## 二、设计原则

### 2.1 维度纯净原则

**核心理念：** 每个页面（含子页面）只承载**单一信息维度**。

**反面案例（绝对避免）：**
- ❌ 在"院内管理"首页同时展示：学生心理预警 + 科研经费执行率 + 项目进度 + 舆情风险
- ❌ 在一个表格中混合：教师科研产出 + 学生论文发表 + 横向项目收入

**正确做法：**
- ✅ "院内管理"拆分为 5 个独立子页面：经费盘点、项目督办、学生事务、舆情安全、中心绩效
- ✅ 每个子页面专注单一主题，数据结构清晰，决策路径明确

**设计验证标准：**
> 院长在任意子页面停留时，脑海中只需思考**一个决策主题**，不会在学生、经费、项目之间跳跃。

---

### 2.2 行动导向原则

**核心理念：** 系统不是信息堆砌器，而是**行动建议引擎**。

每一条呈现给院长的信息都必须回答两个问题：
- **"所以呢？"** —— 这条信息对学院意味着什么（影响评估）
- **"我该做什么？"** —— 具体的行动建议（批示、关注、联络、督办、准备材料）

**对比示例：**

| 传统信息展示 ❌ | 行动导向展示 ✅ |
|---------------|---------------|
| "科技部发布新AI政策" | "科技部AI专项今日17:00截止，匹配度92%，建议立即审批申报材料 [📱催办李主任]" |
| "清华AIR换院长" | "清华AIR新任院长王XX到任，与我院有合作历史，建议本周约见 [💬发祝贺] [📅添加日程]" |
| "学生李某未到实验室" | "学生李某心理预警升至高危，建议今日约谈 [📅添加日程] [📱联系辅导员]" |

**一键操作按钮设计：**
- 所有行动建议都配备**一键执行按钮**
- 钉钉集成：催办、审批、发送、通知
- AI 生成：提纲、推荐信、祝贺信、发言稿

---

### 2.3 渐进披露原则

**核心理念：** 信息分层展示，避免认知过载。

**三层信息架构：**

| 层级 | 展示内容 | 设计目标 |
|------|---------|---------|
| **首页**（院长早报） | Top 3-5 件事 | 3 秒快速扫描，立即决策 |
| **模块首屏** | KPI 概览 + 异常高亮 | 10 秒了解全局，定位问题 |
| **子页面/详情** | 完整数据 + 筛选排序 | 深度分析，下钻细节 |

**设计验证标准：**
> 院长在首页停留 30 秒，应能完成 80% 的日常决策；需要深度分析时才进入子页面。

---

### 2.4 首页是唯一的跨维度聚合页

**核心理念：** "院长早报"是系统中**唯一允许跨维度展示信息的页面**。

**聚合机制：**
- 通过**优先级评分算法**（风险 40% + 紧迫度 30% + 影响 20% + 决策需求 10%）过滤
- 评分 ≥ 60 的事项才浮现到首页
- 最多展示 5 条，避免信息过载

**其他所有页面：** 严格遵循单一维度，不做跨模块聚合

---

### 2.5 同一事件、多维视角原则

**核心理念：** 同一外部事件在不同模块以**不同视角和目的**呈现。

**示例：清华AIR新任院长王XX到任**

| 模块 | 视角 | 展示内容 | 行动按钮 |
|------|------|---------|---------|
| **高校生态** | 竞争情报 | "清华AIR换帅，新院长研究方向：强化学习、机器人。AI分析：该院可能加强产学研合作布局。" | [查看详情] [对标分析] |
| **人脉网络** | 行动触发 | "王XX教授任清华AIR院长，与我院有2019年合作历史，建议本周发祝贺并约见。" | [发送祝贺] [添加日程] |
| **院长早报** | 优先级聚合 | （若优先级≥60）"[人脉·高校] 清华AIR王院长到任，建议本周约见" | [快捷操作] |

**关键差异：**
- **高校生态**：分析竞争对手动向，提供战略建议
- **人脉网络**：关注关系管理，触发社交行动
- **院长早报**：只有优先级够高才出现，提供快捷入口

---

### 2.6 移动优先原则

**核心理念：** 院长 70% 的时间在移动场景，系统必须**移动优先设计**。

**设计策略：**
- 所有核心操作都可在**钉钉**完成，不强制打开PC
- PC 端与钉钉**实时双向同步**（< 2秒延迟）
- 钉钉推送**精准及时**：紧急事项立即推送，常规事项汇总推送

**对比体验：**

| 场景 | 传统系统 ❌ | 本系统 ✅ |
|------|-----------|---------|
| 审批材料 | 必须登录PC查看PDF | 钉钉内直接预览 + 在线审批 |
| 催办任务 | 打电话或发微信 | 一键钉钉催办，自动记录 |
| 查看早报 | 打开电脑登录系统 | 每天8点钉钉自动推送 |
| 添加日程 | 手动输入到日历 | 一键添加，钉钉日历同步 |

---

### 2.7 AI 主动预测原则

**核心理念：** AI 不仅是问答助手，更是**主动预测引擎**。

**主动预测场景：**

| 预测类型 | 触发条件 | 推送内容 | 示例 |
|---------|---------|---------|------|
| **内参机会** | 技术/产业热点事件 | "XX话题适合撰写内参" | "AI监管热点升温，建议撰写政策建议内参" |
| **人才信号** | 学者社交媒体动态 | "XX教授疑似回国意向" | "张教授发文提及'回国发展'，建议主动联络" |
| **关系维护** | 半年未联系重要人士 | "关系冷淡预警" | "李主任（市科委）半年未联络，建议本周问候" |
| **日程冲突** | 新增日程与现有冲突 | "冲突 + 解决方案" | "明日14点两场会议冲突，建议会议B委派副院长" |
| **政策截止** | 申报截止前3天/1天 | "申报倒计时提醒" | "科技部专项明日截止，材料已备需审批" |

|:---:|---------|------------|---------|-------------|
| 1 | **院长早报** | 全维度聚合 | 每日决策入口，只看最重要的事 | 每日8点推送摘要 |
| 2 | **政策情报** | 对国家 + 对北京 | 政策追踪、领导动态、申报机会 | 紧急申报提醒、一键通知责任人 |
| 3 | **科技前沿** | 对技术 + 对产业 | 技术趋势、产业动态、内参选题 | 重大事件推送、一键生成内参 |
| 4 | **人才雷达** | 对人才 | 人才回国追踪、人才指数与排名 | 人才信号通知、一键联络 |
| 5 | **高校生态** | 对高校 | 兄弟单位对标、成果追踪、人事变动 | 重要动态推送 |
| 6 | **院内管理** | 对学院 | 经费/项目/学生/舆情/中心，严格分页 | 异常预警、一键督办、审批流转 |
| 7 | **人脉网络** | 跨维度（人事+关系） | 该联络谁、该祝贺谁、关系维护 | 一键发送祝贺、关系冷淡提醒 |
| 8 | **智能日程** | 对日程 | 日程管理、邀请评估、材料准备 | 日程同步、冲突提醒、材料准备通知 |

**模块设计要点：**

| 设计维度 | 具体指标 |
|---------|---------|
| **模块总数** | 8 个顶层模块 |
| **子页面总数** | 27 个独立子页面 |
| **信息源覆盖** | 75+ 外部信源 + 内部系统数据 |
| **日均推送** | 钉钉推送 8-12 次（可配置） |
| **首页信息量** | Top 3-5 件事（优先级≥60） |
| **一键操作数** | 每个模块平均 5-8 个一键操作按钮 |
| **响应速度** | PC↔钉钉同步 < 2秒 |
| **移动端覆盖** | 100% 核心功能可在钉钉完成 |

### 模块 1：院长早报（首页）

**定位：** 院长每天打开系统的第一个页面。不做信息展开，只做"今天最该关注什么"的高密度摘要。

```

│     建议行动、责任人            │   ─ 风险提示汇总          │
│                                │   ─ 安全指数仪表盘        │
│                                │                         │
├────────────────────────────────┴─────────────────────────┤
│                                                          │
│   维度速览卡片（8 张卡片，每个对应一个模块）                   │
│   ─ 政策情报：新政策 3 条，紧急申报 1 个                     │
│   ─ 科技前沿：热点话题 2 个，融资事件 5 起                   │

└──────────────────────────────────────────────────────────┘
```

**优先级评分机制（决定哪些事项上首页）：**

| 因子 | 权重 | 说明 |
|------|-----|------|
| 风险等级 | 40% | 1=低 2=中 3=高 |
| 时间紧迫度 | 30% | 1=7天+ 2=3天内 3=24小时内 |
| 影响范围 | 20% | 1=个人 2=部门 3=战略级 |
| 需院长决策 | 10% | 是=满分 否=0 |

综合评分 ≥ 60 分的事项浮现到首页，最多展示 5 条。

**安全指数：**
- 满分 100，按风险项扣分
- ≥ 90 绿色，70-89 黄色，< 70 红色
- 展示具体扣分项（类别、扣分值、责任人）

### 5.4 信息实时同步机制

```
┌─────────────┐         ┌─────────────┐         ┌─────────────┐
│  PC端系统   │◄────────│  云端数据库  │────────►│ 钉钉机器人   │

└──────┬──────┘         └──────┬──────┘         └──────┬──────┘
       │                       │                       │
       │ 操作触发               │ 实时同步               │ 推送通知
       │                       │                       │
       ▼                       ▼                       ▼
   数据变更 ──────────────► 云端更新 ──────────────► 多端推送
   (添加待办)              (待办表新增记录)         (钉钉推送)
                                                     (PC端刷新)
```

**同步策略：**

| 数据类型 | 同步方式 | 延迟 |
|---------|---------|------|
| 待办事项 | WebSocket实时推送 | < 1秒 |
| 日程变更 | WebSocket + 钉钉日历API | < 2秒 |
| 审批流转 | 钉钉审批API + 系统回调 | < 3秒 |
| 消息发送 | 钉钉消息API + 系统记录 | < 2秒 |
| 数据查询 | 缓存 + 实时查询 | < 1秒 |
| 报告生成 | 后台任务 + 完成推送 | 10-30秒 |

**冲突解决：**
- PC端和钉钉端同时操作同一待办 → 以最后操作为准
- 离线操作 → 重新联网后自动同步 + 冲突提示

## 六、跨模块交互机制

### 4.1 AI 智能秘书（浮动侧边栏）

AI 秘书是横跨所有模块的对话式助手，具备以下能力：

| 能力 | 说明 |
|------|------|
| **统一行动队列** | 汇聚所有模块中"需要行动"的事项，按优先级排序呈现 |
| **上下文感知** | 根据当前所在模块页面，自动调整对话的上下文和快捷操作 |
| **内容生成** | 一键生成：祝贺信、内参提纲、会议发言稿、汇报材料 |
| **跨模块导航** | 自然语言跳转："带我看看清华最近的动态"→ 跳转到高校生态 |
| **智能问答** | 回答关于任何维度信息的问题 |

### 4.2 跨模块事件关联

同一外部事件可在多个模块中出现，但视角不同：

| 事件示例 | 政策情报中 | 高校生态中 | 人脉网络中 | 院长早报中 |
|---------|-----------|-----------|-----------|-----------|
| 教育部发布新AI人才政策 | **政策原文+申报机会** | — | — | 若优先级≥60则出现 |
| 清华AIR院长换人 | 新院长的政策倾向分析 | **竞争情报：该院战略可能的调整** | **行动：发祝贺消息** | 若优先级≥60则出现 |
| 某AI学者宣布回国 | — | 可能加入的机构分析 | **行动：主动联络** | 若优先级≥60则出现 |
| AI企业完成大额融资 | 相关产业政策关联 | — | 企业负责人关系管理 | 若优先级≥60则出现 |

### 4.3 信息流转规则

```
外部事件产生
    ↓
LLM 分析并分类
    ↓
├── 判定主维度 → 路由到对应模块
├── 判定次维度 → 在其他模块以不同视角展示
├── 判定是否需要行动 → 若需要，推送到「人脉网络」
├── 判定是否有内参价值 → 若有，推送到「内参机会」
└── 计算优先级评分 → 若≥60，浮现到「院长早报」
```

## 十三、常见问题与解答

### Q1: 为什么要将"院内管理"拆分为 5 个子页面？不会增加导航复杂度吗？

**A:** 不会。原因如下：

**问题场景：** 如果不拆分，院长打开"院内管理"会看到：
- 经费执行率 68%
- 项目延期 2 个
- 学生心理预警 1 人
- 舆情负面 3 条
- 中心绩效 5 个

院长的思维会在"钱、项目、人、舆情"之间跳跃，**无法聚焦决策**。

**拆分后的体验：**
- 院长想看经费 → 点击"经费盘点" → 专注思考"钱的问题"
- 院长想看学生 → 点击"学生事务" → 专注思考"学生的问题"

导航增加 1 次点击，但**决策效率提升 300%**。

---

### Q2: 钉钉推送会不会太频繁，打扰院长？

**A:** 我们设计了智能推送策略，避免骚扰：

| 推送类型 | 频率 | 可否关闭 |
|---------|------|---------|
| **紧急事项** | 实时（优先级≥80） | ✗ 不可关闭（院长必须知道） |
| **早报推送** | 每天 8:00 | ✓ 可设置时间 |
| **晚报推送** | 每天 20:00 | ✓ 可关闭 |
| **政策申报倒计时** | 截止前 3天/1天/当天 | ✓ 可调整提前天数 |
| **日程提醒** | 会议前 1天/1小时 | ✓ 可调整时间 |
| **人事变动** | 汇总推送（每天 10:00） | ✓ 可改为实时 |
| **周报** | 每周一 9:00 | ✓ 可关闭 |

**默认配置：** 每天平均 8-10 次推送
**用户可配置：** 在设置中自定义每类推送的时间和频率

---

### Q3: AI 生成的内容（如祝贺信、内参提纲）质量如何保证？

**A:** 多层质量保障机制：

**1. 输入优化：**
- AI 接收**结构化输入**：人物背景、事件细节、历史关系、学院优势等
- 非简单的"生成一封祝贺信"，而是"基于 XX 背景生成 XX 风格的祝贺信"

**2. 模板库：**
- 预设 20+ 常用场景模板（院士当选、职务晋升、论文发表等）
- 院长可自定义模板，AI 学习院长的语言风格

**3. 预览 + 修改：**
- 所有生成内容都**先预览、后确认**
- 一键修改：调整语气、增删内容

**4. 持续学习：**
- 院长每次修改 AI 草稿，系统记录偏好
- AI 模型持续微调，越用越准

**质量对比：**
- 初次使用：80% 可直接使用
- 使用 3 个月后：95% 可直接使用（院长仅需微调细节）

### Q4: 如果院长不在钉钉，只在微信怎么办？

|------|---------|------|

**最佳方案：**

- 备用推送：短信（紧急事项）
- 院长根据场景选择

### Q5: 系统如何防止信息泄露？

**A:** 多层安全机制：

**1. 权限分级：**
```
院长账号: 全部模块全部数据
副院长账号: 部分模块（如不能看学生心理预警细节）
中心主任账号: 仅本中心数据
普通教师账号: 无访问权限
```

**2. 敏感数据脱敏：**
- 学生心理预警：仅显示"某学生"，点击才显示姓名（记录访问日志）
- 财务明细：默认仅显示汇总，明细需二次认证
- 人事变动：非关键人物自动脱敏

**3. 操作日志：**
- 所有查看、审批、发送操作**全程记录**
- 异常访问自动告警（如深夜大量导出数据）

**4. 数据传输：**
- 钉钉通信全程 **HTTPS 加密**
- 敏感字段**二次加密存储**

### Q6: 如果院长出差/休假，系统如何处理待办？

**A:** 智能委派机制：

**1. 出差模式：**
```
院长在系统设置:
  "2024年1月20-25日出差，待办委派给副院长李XX"

系统自动:
  ├ 所有待办推送给李副院长
  ├ 钉钉消息同步发送给李副院长
  ├ 紧急事项（优先级≥90）仍推送院长
  └ 院长回来后，查看委派期间的所有决策记录
```

**2. 休假模式：**
```
院长设置: "春节期间仅推送紧急事项（优先级≥95）"

系统自动:
  ├ 常规事项暂停推送，累积到节后
  ├ 紧急事项立即推送
  └ 节后第一天推送"假期汇总"
```

**3. 自动升级机制：**
- 待办超过 48 小时未处理 → 自动推送给副院长
- 紧急事项超过 6 小时未处理 → 电话 + 短信提醒

- 定期数据校验，确保同步准确性

### 15.4 优先级评分规则速查

**评分公式：**
```
优先级评分 = 风险等级 × 40% + 时间紧迫度 × 30% + 影响范围 × 20% + 需院长决策 × 10%
```

**分值对照表：**

| 因子 | 低 (1分) | 中 (2分) | 高 (3分) |
|------|---------|---------|---------|
| **风险等级** | 低风险 | 中风险 | 高风险 |
| **时间紧迫度** | 7天以上 | 3天内 | 24小时内 |
| **影响范围** | 个人层面 | 部门层面 | 战略层面 |
| **需院长决策** | 否 (0分) | — | 是 (10分) |

**分数段含义：**
- **90-100分：** 红色紧急，立即处理，钉钉脉冲推送
- **70-89分：** 橙色重要，今日处理，钉钉实时推送
- **60-69分：** 黄色关注，本周处理，浮现到早报
- **< 60分：** 不浮现到早报，仅在对应模块展示

**示例：**
```
事项: "科技部AI专项今日17:00截止"
├ 风险等级: 3 (不申报损失重大机会)
├ 时间紧迫度: 3 (24小时内)
├ 影响范围: 3 (战略级项目)
└ 需院长决策: 是 (10分)
计算: 3×40% + 3×30% + 3×20% + 10 = 120 + 90 + 60 + 10 = 280 (归一化后 93分)
结果: 红色紧急，浮现早报首位
```
